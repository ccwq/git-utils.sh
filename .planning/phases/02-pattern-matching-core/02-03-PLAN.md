## Integration Flow

1. cli.py calls load_config() to get alias entries
2. cli.py creates AliasMatcher and adds all aliases
3. cli.py calls get_tokens() on user input
4. cli.py calls matcher.find_best_match(input_tokens)
5. If match found: return (template, captures, rest_capture, args_start)
6. If no match: return None (passthrough)
7. Template expansion comes in Phase 3 (TPL-01 etc)

For now, MATCH-01 and MATCH-02 test that:
- MATCH-01: find_best_match returns correct alias/template
- MATCH-02: args_start indicates where remaining args begin

MATCH-07 (passthrough): If find_best_match returns None, return original input unchanged.
</context>
