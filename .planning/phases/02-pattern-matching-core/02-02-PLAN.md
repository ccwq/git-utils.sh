## Shell Reference Summary

### build_alias_metadata() - L216-267
Creates per-alias metadata:
- alias_count: number of tokens in alias name
- double_token_index: position of ** in alias tokens (-1 if none, -2 if multiple)
- wildcard_weight: 1000 per **, 1 per single *
- literal_chars: non-wildcard characters count
- first_token_mode: "literal" or "wildcard"
- first_token_lower: lowercase first token (for bucket lookup)

Stores in arrays: ALIAS_TOKEN_DATA, ALIAS_TOKEN_COUNTS, ALIAS_DOUBLE_INDEXES, etc.

### find_best_match() - L788-879
Algorithm:
1. Build candidate list from bucket index (first token literal matches + wildcard bucket)
2. For each candidate:
   - Validate token count compatibility
   - Match each alias token against input tokens
   - Track captures and wildcard count
3. Compute score: alias_count * 10000 + literal_chars * 100 - wildcard_weight
4. Select highest scoring match

### Bucket Indexing - L270-288
- ALIAS_BUCKETS_BY_FIRST: dict[first_token_lower] = comma-separated indexes
- ALIAS_BUCKETS_WILDCARD_FIRST: list of indexes where first token has wildcard

4. Track best match: highest score wins
