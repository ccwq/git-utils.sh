## Shell Reference Summary

### get_tokens() - L625-640
- Input: text string
- Output: _TOKENS array (global state)
- Behavior: splits on whitespace, disables glob expansion via `set -f`
- Key: MUST NOT expand globs (unlike shlex.split() which does not, but we need explicit control)

### match_token_pattern() - L648-708
- Input: pattern (e.g., "px*"), token (e.g., "pxhttp-server")
- Output: _MATCH_OK, _MATCH_CAPTURES[], _MATCH_WILDCARDS (globals)
- Behavior:
  - No wildcards: case-insensitive string comparison
  - Has wildcards: split by *, escape parts, build regex with (.*?) groups
  - Replace (.*?) with (.*) for greedy behavior (bash =~ doesn't support lazy)
- D-17: Use fnmatch.translate() but manually handle greedy

### match_double_star_remainder() - L716-775
- Input: pattern (e.g., "s**"), input_text (e.g., "ls -l")
- Output: _DSTAR_OK, _DSTAR_CAPTURES[], _DSTAR_REST (globals)
- Behavior:
  - Step 1: Find ** position, split into head and tail
  - Step 2: Build regex for head+tail, match against input
  - Step 3: String slice to extract remainder (not captured directly by regex)
- D-18: Two-step match with remainder extraction
</context>

Add match_double_star_remainder() function to py/wsha/matching.py.

Implementation per D-18 (two-step match):

1. Find ** position in pattern:
   - If no ** found, return failure
   - Split into head (before **) and tail (after **)

2. Build regex for head+tail matching:
   - Escape special regex chars in head/tail
   - Replace * in head/tail with (.*?) then convert to greedy (.*)
   - Build: ^head_regex(.*?)tail_regex$
   - Note: The middle (.*?) captures the remainder

3. Match against input_text using case-insensitive re.search()

4. Extract captures:
   - Groups before last are head captures
   - Last group is the ** remainder

Return (match_ok: bool, head_captures: list[str], rest_capture: str)

Reference: sh/wsha.sh L716-775
